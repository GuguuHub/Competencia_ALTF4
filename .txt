--------------------------------------------------
1. OBJETIVO DEL PROGRAMA
--------------------------------------------------

El objetivo del programa es simular el sistema de clasificación
de la fase de grupos del Mundial FIFA 2026.

El sistema recibe los resultados de los partidos de un grupo
compuesto por 4 equipos y debe:

- Registrar estadísticas de cada equipo
- Calcular puntos
- Calcular goles a favor y en contra
- Calcular diferencia de gol
- Ordenar la tabla según criterios FIFA
- Determinar clasificados
- Identificar el tercer puesto
- Manejar casos especiales y validaciones

El programa fue desarrollado utilizando Python y estructuras
de datos dinámicas como diccionarios, listas, sets y funciones.

3.1 FUNCIÓN imprimir_tabla()
--------------------------------------------------

Esta función se encarga de mostrar la tabla de posiciones
de manera ordenada y visual.

Parámetros:

- tabla → lista de equipos ordenados
- titulo → nombre de la tabla a mostrar

Se utiliza:

print(f"{' POS':<5}")

El símbolo "<5" significa:
alinear el texto a la izquierda utilizando
5 espacios.

La función también utiliza:

enumerate(tabla, 1)

Esto permite recorrer la tabla y numerar automáticamente
las posiciones comenzando desde 1.

Ejemplo:
1 ARG
2 BRA

Luego se calcula la diferencia de gol:

dg = datos["gf"] - datos["gc"]

Finalmente se imprime toda la tabla utilizando
f-strings para mantener el formato alineado.

Esta función verifica si un equipo ya existe dentro

if equipo not in equipos:

--------------------------------------------------
3.3 FUNCIÓN validar_equipo_clasificado()
--------------------------------------------------

Esta función valida que el país ingresado pertenezca
a la lista de selecciones clasificadas al Mundial 2026.

Se utiliza:

clave = nombre.casefold()

casefold() convierte el texto a minúsculas
de forma segura.

Ejemplo:

"ARG" → "arg"

Esto permite validar correctamente el texto
sin importar mayúsculas o minúsculas.

Luego se verifica:

if clave not in equipos_validos_map:

Si el país no existe:
se genera un error mediante:

raise ValueError

Posteriormente ese error es capturado
por un bloque try/except.

--------------------------------------------------
3.5 FUNCIÓN procesar_partido()
--------------------------------------------------

Esta función actualiza todas las estadísticas
luego de disputarse un partido.

Se modifican:

- partidos jugados
- goles a favor
- goles en contra
- puntos

Ejemplo:

equipos[local]["gf"] += gl

Esto significa:
sumar los goles realizados por el local.

También se actualizan los puntos.

Sistema utilizado:

- victoria → 3 puntos
- empate → 1 punto
- derrota → 0 puntos

Condiciones:

if gl > gv:
    gana local

elif gl < gv:
    gana visitante

else:
    empate

--------------------------------------------------
3.6 FUNCIÓN ordenar_tabla()
--------------------------------------------------

Esta función ordena los equipos utilizando
los criterios oficiales FIFA.

Se utiliza:

tabla.sort()

junto con:

key=lambda x:

La función lambda define el criterio de ordenamiento.

Código:

tabla.sort(
    key=lambda x: (
        -x[1]["pts"],
        -(x[1]["gf"] - x[1]["gc"]),
        -x[1]["gf"],
        x[0]
    )
)

Explicación:

- x[0] → nombre del equipo
- x[1] → estadísticas del equipo

Orden utilizado:

1. Mayor cantidad de puntos
2. Mayor diferencia de gol
3. Mayor cantidad de goles a favor
4. Orden alfabético

El signo negativo (-) permite ordenar
de mayor a menor.

--------------------------------------------------
4. CORNER CASES CONTEMPLADOS
--------------------------------------------------

El programa fue diseñado considerando
múltiples situaciones límite.

1) Entrada vacía

Si el usuario presiona ENTER sin ingresar datos,
el sistema muestra un error y vuelve a pedir
el partido.

2) Cantidad incorrecta de datos

Cada línea debe tener exactamente:

EquipoLocal EquipoVisitante GolesL GolesV

Si faltan o sobran datos,
el sistema invalida la entrada.

3) País no clasificado

Solo se permiten selecciones clasificadas
al Mundial FIFA 2026.

4) Equipo contra sí mismo

Un equipo no puede jugar contra sí mismo.

Ejemplo inválido:

ARG ARG 2 1

5) Partido repetido

No se permite repetir enfrentamientos.

Ejemplo:

ARG BRA
BRA ARG

Se consideran el mismo partido.

6) Goles fuera de rango

Los goles deben estar entre 0 y 20.

7) Más de 4 equipos

El sistema controla que el grupo
tenga exactamente 4 selecciones.

8) Más de 3 partidos por equipo

Cada selección solo puede disputar
3 encuentros.

9) Empate absoluto

Si dos equipos empatan en:

- puntos
- diferencia de gol
- goles a favor

se utiliza orden alfabético.

10) Error de tipo de dato

Si el usuario ingresa texto en lugar de números
para los goles,
el programa captura el error utilizando
try/except.
